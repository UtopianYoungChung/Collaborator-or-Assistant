# Replication Package: Collaborator or Assistant?

## Human-AI Collaboration Across Pull Request Lifecycles

This replication package supports the paper **"Collaborator or Assistant? How AI Coding Agents Partition Work Across Pull Request Lifecycles"** (AIWare 2026). It provides the data, code, and documentation required to reproduce the analysis and figures.

## Prerequisites

- **Python 3.9+**
- **Raw data**: Five PR timeline JSON files in `data/raw/` (see `DATA_ACCESS.md`)
- **Disk space**: ~850 MB for raw data; ~50 MB for generated artifacts

---

## Quick Start

**1. Clone or extract this replication package** into the AIWare project root (the folder containing `src/`, `data/`, `config/`).

**2. Install dependencies:**
```bash
pip install -r replication_package/requirements.txt
```
Optional: Install [Graphviz](https://graphviz.org/) for higher-quality state machine PDFs (the pipeline falls back to matplotlib if graphviz is unavailable).

**3. Ensure raw data is in place** (see `DATA_ACCESS.md`):
```
data/raw/pr_timelines_Claude_Code.json
data/raw/pr_timelines_Copilot.json
data/raw/pr_timelines_Cursor.json
data/raw/pr_timelines_Devin.json
data/raw/pr_timelines_OpenAI_Codex.json
```

**4. Run the replication pipeline** from the project root:

**Windows:**
```cmd
replication_package\run_replication.bat
```

**Unix/macOS:**
```bash
bash replication_package/run_replication.sh
```

Or run the steps manually (see below).

## Pipeline Steps (Manual)

Run these commands from the **project root** (the directory containing `src/` and `data/`):

```bash
# Step 1: Generate evidence statistics (collaboration-type distribution, χ², Cramér's V)
python -m src.analysis.evidence_stats --out-json .tmp/evidence_stats.json --out-md .tmp/evidence_stats.md

# Step 2: Compute phase transition probabilities and median hours per tool
python -m src.analysis.network_extraction --output-dir .tmp/networks --exclude-incomplete

# Step 3: Within-tool comparison (human-initiated PRs only; construct validity rates)
python -m src.analysis.within_tool_comparison

# Step 4: Temporal analysis (94% from 2025 Q2–Q3 concentration)
python -m src.analysis.temporal_trends --out-json .tmp/temporal_analysis.json

# Step 5: Downsampling robustness (Threats: OpenAI downsampled; Cramér's V stability)
python -m src.analysis.downsample_robustness --n-match 4885 --seeds 10 --output .tmp/downsample_robustness.json

# Step 6: Within-repository control (Discussion / Threats: initiation gap vs review routing)
python -m src.analysis.within_repository_analysis --min-prs 5 --repo-rule all_five_tools --out-json .tmp/within_repository_analysis.json --out-md .tmp/within_repository_analysis.md

# Step 7: Generate the collaboration distribution figure (Fig. 3 in paper)
python -m src.analysis.plot_collaboration_distribution

# Step 8: Generate the five state machine figures (Figs 4–8 in paper)
python -m src.analysis.plot_state_machines
```

**Output locations:**
- `.tmp/evidence_stats.json`, `.tmp/evidence_stats.md` — Deterministic statistics (tables, χ², Cramér's V)
- `.tmp/networks/phase_transition_probabilities_by_tool.md` — Phase transition P(To|From) and median hours
- `.tmp/networks/s2_path_analysis.md` — RQ3: exhaustive path classification for Agent-merge scenarios (S2/S5)
- `.tmp/within_tool_human_initiated_report.json`, `.tmp/within_tool_human_initiated_report.md` — Construct validity: first exit from PR created (human-init only) with 95% Wilson CIs, e.g. Copilot 87.8%→review [75.8%, 94.3%], OpenAI 11.1%→review [10.7%, 11.6%]
- `.tmp/temporal_analysis.json` — Temporal distribution; 94% of PRs from 2025 Q2–Q3
- `.tmp/downsample_robustness.json` — Per-seed Cramér's V after downsampling OpenAI to Copilot size ($n{=}4885$)
- `.tmp/within_repository_analysis.json`, `.tmp/within_repository_analysis.md` — Repos with both paradigms ($\geq$5 PRs each); initiation gap and human-merge-actor shares within repo
- `AIWare2026_CameraReady_Package/figures/fig_collab_distribution.pdf` — Collaboration scenario stacked bar chart (Fig. 3)
- `AIWare2026_CameraReady_Package/figures/fig_state_machine_*.pdf` — Five per-tool state machine diagrams (Figs 4–8)

**Figures not produced by this pipeline:** `fig_pipeline_overview.pdf` and `fig_pr_workflow.pdf` in the camera-ready PDF are supplied separately (or generated by project diagram tools); reproduce statistics from the steps above, not from those diagrams.

## Expected Results

- **Total PRs (raw):** 33,600  
- **Included PRs (6 scenarios, terminal outcomes):** 29,585  
- **Tool × collaboration-type association:** Cramér's V ≈ 0.50–0.51  
- **Temporal concentration:** ~94% of PRs from 2025 Q2–Q3 (verifiable from `.tmp/temporal_analysis.json`)
- **Construct validity (human-init only):** Copilot 87.8%→review [75.8%, 94.3%], Devin 61.0%→review [49.9%, 71.2%], Cursor 57.1%→review [44.1%, 69.2%], OpenAI 11.1%→review [10.7%, 11.6%]; non-overlapping Wilson 95% CIs between paradigms. Verifiable from `.tmp/within_tool_human_initiated_report.md`
- **Downsampling robustness:** Cramér's V stable near 0.51 with OpenAI downsampled to $n{=}4885$ across 10 seeds (`.tmp/downsample_robustness.json`)
- **Within-repository control:** With `--repo-rule all_five_tools` (each tool ≥5 included PRs per repo), camera-ready reports 34 repositories and 2,197 PRs; initiation gap $\approx$96.8 pp and human-merge-actor reversal (Assistant vs Collaborator) on the **paper’s operationalization**. If this rule yields **zero** repos on a sparse extract, use `--repo-rule both_paradigms` for diagnostics (aggregate paradigm counts per repo). Pool-level human-resolver shares are in `.tmp/within_repository_analysis.*`.
- **RQ3 paths:** Cross-check automation-authorized merge narrative with `.tmp/networks/s2_path_analysis.md` and RQ3 section in the paper
- **Figures:** 1 collaboration distribution PDF (`AIWare2026_CameraReady_Package/figures/fig_collab_distribution.pdf`); 5 state machine PDFs (`AIWare2026_CameraReady_Package/figures/fig_state_machine_*.pdf`)

Verification: Compare `.tmp/evidence_stats.md`, `.tmp/within_tool_human_initiated_report.md`, `.tmp/temporal_analysis.json`, `.tmp/downsample_robustness.json`, `.tmp/within_repository_analysis.md`, `.tmp/networks/s2_path_analysis.md`, and the generated figures to the values and visuals in the paper. See `SOURCE_OF_TRUTH.md` for manuscript-to-artifact mapping.
